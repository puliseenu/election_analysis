DATABASE STRUCTURE
===============================================================================

USERS TABLE:
  • id (Primary Key)
  • username (Unique) - Login name
  • email (Unique) - Contact email
  • password_hash - Encrypted password (PBKDF2-SHA256)
  • role - 'admin' or 'user'
  • status - 'pending', 'approved', or 'rejected'
  • created_at - Registration date/time
  • approved_at - Approval date/time
  • approved_by - Admin who approved
  • last_login - Last login timestamp
  • is_active - 1 (active) or 0 (inactive)

AUDIT_LOG TABLE:
  • id (Primary Key)
  • action - Type of action (login, register, approve, etc.)
  • user_id - User performing action
  • username - Username performing action
  • details - Description of action
  • ip_address - Source IP address
  • timestamp - When action occurred

SESSIONS TABLE:
  • id (Primary Key)
  • user_id - User session belongs to
  • token - Session token
  • created_at - Session start time
  • expires_at - Session expiration
  • ip_address - Client IP
  • user_agent - Browser info
  • is_active - 1 (valid) or 0 (expired/revoked)

  - PBKDF2-SHA256 hashing

✓ ENCRYPTION
  - Passwords encrypted with PBKDF2-SHA256

  - Session tokens stored in database

USER WORKFLOW
===============================================================================

NEW USER:
  1. Go to login page
  2. Click "REGISTER"
  3. Enter username, email, password
  4. Click "REGISTER"
  5. See "Registration successful! Awaiting admin approval"

  7. Admin approves in PENDING USERS

EXISTING USER:
  1. Go to login page
  2. Enter username and password
  3. Click "LOGIN"
  4. Dashboard opens (analytics)

  6. Click "LOGOUT" to exit

ADMIN:
  1. Login with admin credentials
  2. See ADMIN PANEL
  3. PENDING USERS tab shows new registrations
  4. Click "APPROVE" button for each user

===============================================================================
TROUBLESHOOTING
===============================================================================

Problem: "Username or password incorrect"
Solution:
  - Check username spelling
  - Check for CAPS LOCK
  - Verify account is approved (not pending)
  - Reset admin: delete auth_database.db, restart

Problem: "Username already exists"
Solution:
  - Choose different username
  - Contact admin if needed

Problem: "Account pending approval"
Solution:
  - Admin needs to approve in PENDING USERS
  - Check back after approval

Problem: Cannot access 192.168.0.165:8050
Solution:
  - Check network connection
  - Both devices on same WiFi?
  - Firewall blocking?
  - Dashboard running?

Problem: "Database locked"
Solution:
  - Stop all Python processes
  - taskkill /F /IM python.exe
  - Wait 2 seconds
  - Restart dashboard

Problem: Port 8050 in use
Solution:
  - Stop other processes: taskkill /F /IM python.exe
  - Or change port in dashboard_auth.py last line
  - Restart dashboard

Problem: Lost admin password
Solution:
  - Delete auth_database.db
  - Restart dashboard
  - Admin account recreated (admin/admin1432)
