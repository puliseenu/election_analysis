👥 USER WORKFLOW

NEW USER REGISTRATION:
  1. Click "REGISTER" on login page
  2. Fill: username, email, password
  3. Status: "Awaiting admin approval"
  4. Admin approves in PENDING USERS
  5. User can login immediately

EXISTING USER LOGIN:
  1. Enter username & password
  2. Click LOGIN
  3. Dashboard opens
  4. View analytics & data
  5. Click LOGOUT to exit

ADMIN WORKFLOW:
  1. Login with admin credentials
  2. See ADMIN PANEL
  3. Review PENDING USERS
  4. Click APPROVE or REJECT
  5. Approved users can login
  6. LOGOUT to exit

📊 DATABASE STRUCTURE

USERS TABLE:
  • id, username (unique), email (unique)
  • password_hash (encrypted)
  • role (admin/user), status (pending/approved/rejected)
  • created_at, approved_at, approved_by
  • last_login, is_active

AUDIT_LOG TABLE:
  • action, user_id, username
  • details, ip_address, timestamp

SESSIONS TABLE:
  • user_id, token (unique)
  • created_at, expires_at
  • ip_address, user_agent, is_active

🆘 TROUBLESHOOTING

Issue: Cannot login
  → Check username/password spelling
  → Verify account is approved (not pending)
  → Try resetting admin (delete auth_database.db)

Issue: Port 8050 in use
  → taskkill /F /IM python.exe
  → Wait 2 seconds
  → Restart dashboard

Issue: Database locked
  → Stop all Python processes
  → Restart dashboard

Issue: Lost admin password
  → Delete auth_database.db
  → Restart (recreates admin)
  → Login: admin / admin1432
